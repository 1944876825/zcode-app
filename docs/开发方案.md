# ZCode 移动端 APP 开发方案 (全原生)

## 一、项目概述

将 zcode 网页端 AI 编程助手平台转化为 **完全原生** Flutter 移动应用。

zcode 是智谱AI (bigmodel.cn) 的 AI 编程助手,采用 Relay 中继远程控制架构:
APP 作为瘦客户端,通过 WebSocket + REST API 连接 Relay 服务器,
远程环境负责 AI 推理和文件操作,APP 负责交互渲染。

## 二、逆向分析结论 (详见 API逆向分析.md)

| 维度 | 结论 |
|------|------|
| 架构 | Relay 中继,客户端 = 瘦客户端 |
| 连接 | WebSocket (长连接事件流) + HTTP REST |
| 认证 | OAuth token + remoteControlToken + relayOrigin |
| 协议 | JSON 消息,zcode_type 字段标识类型 |
| 流式 | WebSocket 推送 agent_message_chunk 等 |
| 移动端 | 后端已有 mobile-view-state API,已考虑移动端 |
| 模型 | GLM-5.2,走 OpenAI chat-completions 格式 |

## 三、技术栈

| 领域 | 技术 | 说明 |
|------|------|------|
| 框架 | Flutter 3.44.2 + Dart 3.12 | 已安装 |
| 状态管理 | Riverpod 2.x | 类型安全,支持流式数据 |
| 路由 | go_router | 声明式路由 |

| HTTP | Dio | REST API 调用 |

| 代码高亮 | re_highlight | 代码块语法高亮 |

| 图标 | Material Symbols | Material 3 |

### 4.2 Relay 协议客户端 (核心)

这是整个 APP 最关键的组件 —— 用 Dart 实现 zcode 的 Relay 通信协议:

```
RelayClient
├── 连接管理
│   ├── connect(token, relayOrigin)    建立连接
│   ├── bootstrap()                     引导握手,获取工作区列表
│   ├── openWorkspaceBridge(wsKey)     打开工作区桥接
│   ├── reconnect()                     断线重连
│   └── heartbeat()                     心跳保活
│
├── 消息收发
│   ├── send(zcode_type, payload)       发送消息
│   ├── onMessage.stream                消息流 (供 Provider 监听)
│   ├── requestResponse(type, payload)  请求-响应配对
│   └── requestId 生成与管理
│
├── 事件分发
│   ├── agentEvents.stream              AI agent 事件流
│   ├── taskEvents.stream               任务生命周期事件
│   ├── fileEvents.stream               文件变更事件
│   └── errorEvents.stream              错误事件
│
└── REST 调用
    ├── platformRPC(method, args)       平台 RPC
    ├── getWorkspaces()                 工作区列表
    └── updateMobileViewState(state)    移动端视图状态
```

## 五、核心交互设计

### 5.1 导航结构

```
[Splash] → 检查 Token
  ├─ 无 Token → [登录] → [验证码] → [OAuth回调] → 获取 remoteControlToken
  └─ 有 Token → [主页]
                   │
                   ├─ 工作区 Tab
                   │   ├─ 项目列表
                   │   └─ → [对话页] (核心)
                   │
                   ├─ 技能 Tab
                   │   └─ 技能列表/管理
                   │
                   └─ 我的 Tab
                       ├─ 额度
                       ├─ 模型设置
                       ├─ 主题
                       └─ 关于
```

### 5.2 AI 对话页 (核心页面)

```
┌─────────────────────────────┐
│ ☰  rrsp / 采集功能改多线程  ⋮ │  ← 顶栏: 返回 + 项目/任务 + 菜单
├─────────────────────────────┤
│                             │
│  [用户] 帮我把采集改为多线程  │  ← 消息列表 (反向滚动)
│                             │
│  [AI] 好的,我来分析一下...   │  ← Markdown 渲染
│  ```python                  │  ← 代码块 (高亮 + 复制)
│  from concurrent.futures    │
│  import ThreadPoolExecutor  │
│  ```                        │
│                             │
│  [差异] collector.py +12 -3 │  ← 代码差异 (可展开)
│  [AI正在输入...]            │  ← 流式输入动画
│                             │
├─────────────────────────────┤
│ @文件  /命令  [变更前确认▾]  │  ← 快捷操作栏
│ ┌─────────────────────────┐ │
│ │ 输入消息...          [➤] │ │  ← 输入框 + 发送
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

### 5.3 流式响应处理流程

```
用户发送消息
  ↓
ChatProvider.sendMessage(text)
  ↓
RelayClient.send(platform-request, { method: "chat", args: [...] })
  ↓
WebSocket 接收事件流:
  ├─ task_run_started      → UI: 显示 "AI 正在工作"
  ├─ agent_thought_chunk   → UI: 显示思考过程 (折叠区)
  ├─ agent_message_chunk   → UI: 逐字追加到消息气泡
  ├─ diff                  → UI: 显示代码差异卡片
  ├─ task_token_usage_delta→ UI: 更新 Token 用量
  └─ task_complete         → UI: 停止动画,消息完成
```

## 七、关键技术难点与对策

### 7.1 Relay 协议实现 (最大难点)
问题: 需要用 Dart 完整实现 zcode 私有协议
对策:
1. 先用 mitmproxy 抓包记录完整的连接→对话→断开流程
2. 按消息类型逐个实现,先跑通 bootstrap,再逐步加功能
3. 参考已逆向的消息类型表 (API逆向分析.md 第四节)

### 7.2 登录认证 (验证码)
问题: zcode 使用阿里云验证码,纯原生较难复现
对策:
1. 分析 OAuth 流程,看是否有 API Key / Token 直接登录方式
2. 首版可让用户从网页端复制 session URL (sid+hash) 导入
3. 后续集成阿里云验证码 SDK 或找替代认证方式

### 7.3 流式渲染性能
问题: 长对话 + 流式追加可能导致卡顿
对策:
1. 使用 ListView.builder 虚拟化
2. 流式文本用 TextSpan 增量更新,不重建整个 Widget
2. 代码块单独组件,延迟解析高亮
4. 限制单条消息最大长度,超长分段

### 7.4 WebSocket 稳定性
问题: 移动网络不稳定,WebSocket 易断开
对策:
1. 实现指数退避重连
2. 心跳保活 (定时 ping)
3. 断线期间消息排队,重连后补发
4. 网络状态监听,有网自动重连
